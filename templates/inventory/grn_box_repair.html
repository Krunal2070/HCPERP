Why this page exists

When a GRN is saved, the system creates one rm_boxes row per package. If something goes wrong (e.g. the material's name in the GRN line doesn't exactly match the materials table), the GRN save still reports "success" but the rows are never created — the printed labels then fail when scanned at Stock Transfer with "No package with code RM-A…".

This page lists every GRN line with that gap and lets you create the missing rows using the exact codes already printed on your labels. After repair, existing stickers will scan correctly — no re-printing required.
